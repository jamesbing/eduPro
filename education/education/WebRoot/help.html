Cache Address Bit Pattern and Cache Structure

This document shows what each of the entries in the simulator expects giving a suitable example.

Details of options

1)  Memory Size
This figure determines the total number of bits in the address bit pattern.
Alternately, it can be replaced by the variable - total number of bits in the physical address.

2) Cache Size
This option determines the size of the cache in Bytes

3) Block Size
Main Memory consists of blocks. Each block consists of 1 or more bytes. The number of bytes in each block determines the number of offset bits

4) Cache Scheme
The user can choose the type of mapping that is used in the cache. User can choose between Direct and Set Associative Mapping
Set Size allows the user to choose the number of blocks in a set
